# Data Dictionary

This project uses the Maven Analytics **CRM Sales Opportunities** dataset
(fictitious B2B hardware company "MavenTech") as its raw source, plus one
synthesized table — `subscriptions` — built specifically for this project
to add post-sale metrics (renewals, expansion, churn) that the original
dataset does not include.

---

## Source tables (Maven Analytics, unmodified)

### `accounts`
| Field | Description |
|---|---|
| account | Company name |
| sector | Industry |
| year_established | Year established |
| revenue | Annual revenue (in millions of USD) |
| employees | Number of employees |
| office_location | Headquarters |
| subsidiary_of | Parent company (if applicable) |

### `products`
| Field | Description |
|---|---|
| product | Product name |
| series | Product series |
| sales_price | Suggested retail price |

### `sales_teams`
| Field | Description |
|---|---|
| sales_agent | Sales agent name |
| manager | Respective sales manager |
| regional_office | Regional office |

### `sales_pipeline`
| Field | Description |
|---|---|
| opportunity_id | Unique identifier |
| sales_agent | Sales agent |
| product | Product name |
| account | Company name |
| deal_stage | Sales pipeline stage (Prospecting → Engaging → Won / Lost) |
| engage_date | Date the "Engaging" deal stage was initiated |
| close_date | Date the deal was "Won" or "Lost" |
| close_value | Revenue from the deal |

**Known data quality notes:**
- `engage_date` and `close_date` are null for opportunities that never left the Prospecting stage or are still open.
- Row count: 8,800 opportunities total; 4,238 reached "Won."

## Synthesized table: `subscriptions`

**This table does not exist in the original Maven dataset.** The source
data covers pre-sale pipeline only (Prospecting through Won/Lost) with no
information on what happens after a deal closes. Since the target job
description (Planful Sales Ops Analyst) requires post-sale metrics — NRR,
GRR, churn, expansion — this table was generated to model a plausible
12-month subscription lifecycle on top of every "Won" opportunity.

Generated by `generate_postsale.py`, seeded with `random.seed(42)` for
full reproducibility.

| Field | Description |
|---|---|
| subscription_id | Synthetic unique identifier (`SUB-00001`, etc.) |
| opportunity_id | Foreign key back to `sales_pipeline.opportunity_id` |
| account | Company name (foreign key to `accounts.account`) |
| product | Product name (foreign key to `products.product`) |
| sales_agent | Sales agent who closed the original deal |
| start_date | Subscription start date = the original `close_date` |
| renewal_date | `start_date` + 12 months |
| starting_arr | First-year ARR = the original `close_value` |
| renewal_outcome | One of: `Renewed Flat`, `Expansion`, `Churned`, `Contraction` |
| renewal_arr | ARR at renewal, after applying the outcome multiplier |

### Renewal outcome logic

Each Won opportunity is assigned a renewal outcome using a fixed
probability split, chosen to reflect a realistic mid-market B2B SaaS
retention profile:

| Outcome | Probability | ARR multiplier | Meaning |
|---|---|---|---|
| Renewed Flat | 70% | 1.0x | Customer renews at the same ARR |
| Expansion | 15% | random 1.10x – 1.40x | Upsell / cross-sell — customer grows |
| Churned | 10% | 0x | Customer does not renew |
| Contraction | 5% | random 0.60x – 0.90x | Customer downgrades but stays |

### Resulting metrics (from actual run on the full 4,238-record dataset)

| Metric | Value |
|---|---|
| Gross Revenue Retention (GRR) | 89.4% |
| Net Revenue Retention (NRR) | 93.3% |
| Logo churn rate | 9.1% |
| Total starting ARR | $10,005,534 |
| Total renewal ARR | $9,332,402 |

**Why NRR is 93.3% and not 100%+:** with 10% of accounts churning to $0
and only 15% expanding, the math works out below the "best-in-class SaaS"
100%+ NRR benchmark. This is intentional — a coherent, realistic profile
for a mid-market hardware/software seller with healthy retention but real
expansion headroom, rather than an artificially perfect number. The
underlying math: `(0.70 × 1.0) + (0.155 × ~1.25) + (0.091 × 0) + (0.051 × ~0.75) ≈ 93.2%`,
which matches the actual generated output almost exactly.

### Important caveat for anyone reviewing this project

The `subscriptions` table is a **modeled simulation**, not real customer
data. It exists to demonstrate SQL, BI, and analytical technique on
post-sale SaaS metrics using a dataset that otherwise doesn't include
them. This is documented transparently here and in the project README
rather than presented as if it were part of the original source data.
